# Колыбель — рабочее пространство (с личными кабинетами)

## ⚠️ Важно: способ публикации изменился

Раньше мы просто перетаскивали папку в Netlify Drop. **Теперь это не сработает** — у сайта появилась
серверная часть (Functions + хранилище данных), и для её работы Netlify должен установить зависимости
(`npm install`) при публикации. Drag-and-drop этого не делает.

### Как опубликовать (рекомендуемый способ — через GitHub)

1. Создайте бесплатный аккаунт на **github.com**, если его нет.
2. Создайте новый репозиторий (например `kolybel-internal`), загрузите туда всю папку через
   веб-интерфейс GitHub («Add file» → «Upload files» → перетащите всё содержимое папки).
3. В Netlify: **Add new site → Import an existing project → Deploy with GitHub** → выберите этот репозиторий.
4. Netlify сам найдёт `netlify.toml`, установит зависимости и опубликует сайт. Готово.

### Альтернатива — через Netlify CLI (если не хочется заводить GitHub)

1. Установите Node.js (nodejs.org), затем в терминале: `npm install -g netlify-cli`
2. В папке проекта: `npm install` (установит зависимости функций)
3. `netlify deploy --prod` → следуйте подсказкам (вход в Netlify, выбор/создание сайта)

## Вход в систему — 3 аккаунта

| Роль | Логин | Пароль |
|---|---|---|
| Менеджер (Валерия) | `valeria` | `Valeria2026!` |
| Менеджер 2 | `manager2` | `Manager2026!` |
| Директор | `director` | `Director2026!` |

**Обязательно смените пароли** перед реальным использованием — это сделать просто:
в Netlify: **Site settings → Environment variables**, добавьте:

```
VALERIA_USER, VALERIA_PASS
MANAGER2_USER, MANAGER2_PASS
DIRECTOR_USER, DIRECTOR_PASS
AUTH_SECRET   (любая длинная случайная строка — секрет для подписи сессий)
```

После изменения переменных нажмите **Trigger deploy** в Netlify, чтобы изменения применились.

Если хотите переименовать «Менеджер 2» — напишите мне настоящее имя, я поправлю в коде.

## Что внутри

- **login.html** — вход.
- **manager-dashboard.html** — кабинет менеджера: мини-CRM (клиент / услуга / сумма / няня / стадия) +
  календарь звонков с уведомлением за 30 минут.
- **director-dashboard.html** — кабинет директора: видит сделки всех менеджеров, авторасчёт ЗП
  (20% от суммы закрытых сделок, выплата раз в 3 месяца — это вы делаете сами по графику, цифры
  считаются автоматически), плюс журнал доходов/расходов (замена гугл-таблицы).
- **standard.html**, **anketa-builder.html** — то, что уже было, теперь тоже требует входа.
- Данные хранятся в **Netlify Blobs** — это встроенное хранилище Netlify, отдельный аккаунт заводить
  не нужно, оно появляется автоматически при публикации сайта.

## Push-уведомления о звонках (за 30 минут)

Работает через стандартный Web Push — тоже без сторонних аккаунтов. После публикации:
1. Менеджер заходит в свой кабинет с **телефона**, добавляет сайт на главный экран (как обсуждали ранее).
2. При первом заходе появится баннер «Включить уведомления» — нужно нажать и разрешить уведомления в браузере.
3. Дальше уведомления приходят автоматически, даже если приложение закрыто.

Технический момент: проверка предстоящих звонков запускается раз в 5 минут (Netlify Scheduled Function).
Это может требовать платный тариф Netlify для частого запуска — если после публикации увидите, что
функция `check-reminders` не запускается по расписанию, напишите мне, разберёмся с тарифом.

## Честный момент про безопасность

Это простая, но рабочая система входа (подписанные сессионные cookie), подходящая для внутреннего
небольшого инструмента. Это не банковский уровень защиты — для команды из 3 человек этого достаточно,
но если данные станут более чувствительными (например, паспортные данные клиентов), стоит обсудить
усиление защиты отдельно.
